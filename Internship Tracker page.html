Internship Tracker

Web application — July 2025 – Present

Project Description

Internship Tracker is a web app that helps organize internship and job applications in one place. Instead of manually copying details from each posting, users paste a link and let the app pull out the key information, then track status, notes, and documents for every application.

Why I Built This

Keeping track of multiple internships and job applications across different sites quickly gets messy. I wanted a single dashboard where I could store links, role details, deadlines, and my own notes—and automate as much of the data entry as possible. This project was also a chance to work with scraping tools, headless browsers, and LLMs in a practical way.

Key Features

  • Paste a job or internship posting URL and automatically extract the role title, company, location, and description.
  • Dashboard view to track each application with fields for status, notes, and next actions.
  • Attach resumes and cover letters to each application for quick reference.
  • Filter or sort applications by status (e.g., applied, interviewing, rejected, offer).

How It Works

When a user submits a job posting URL, the backend launches Puppeteer to load the page and capture its HTML. That content is parsed with Cheerio to extract structured pieces like title, company, and location where possible. The full content and any partially structured data are then sent to OpenRouter AI, which returns a normalized JSON payload containing the main fields the dashboard needs.

The frontend is built in Next.js, providing a modern React-based experience with server-side or API routes for the scraping and AI calls. Application records, statuses, and file metadata are saved so that users can come back and continue tracking their search over time.

Technologies Used

  • Next.js (React framework for the frontend and API routes)
  • Puppeteer (headless Chrome for loading and scraping pages)
  • Cheerio (server-side HTML parsing and extraction)
  • OpenRouter AI (LLM-powered extraction and normalization)
  • JavaScript/TypeScript
  • GitHub (version control and collaboration)

Challenges & Lessons Learned

  • Handling different job board layouts and making the scraper resilient to small HTML changes.
  • Designing prompts and response schemas for the AI so that output is reliable enough to store directly.
  • Coordinating browser automation, parsing, and AI calls without blocking the UI or making users wait too long.
  • Structuring the data model to support status changes, notes, and file attachments in a clean way.
